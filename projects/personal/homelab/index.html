Homelab Infrastructure & Self-Hosted Services

A old gaming computer turned into a fully functioning home server with local LLM, SMB file storage, and secure access

Overview

The homelab itself features one server built from a old gaming computer. The hypervisor itself is proxmox based with various LXCs for various services. This setup was created with a security first mindset so this includes things like a firewall, vpn only access, isolation of containers, etc.

Architecture & Tech Stack

  • Hypervisor: Proxmox
  • Firewall: OPNSense
  • Networking: Traefik as a reverse proxy, paired with Technitium DNS for local domain resolution. Remote access is secured via Tailscale
  • Orchestration: Docker & Docker Compose

Key Features

Unified Dashboard

To monitor the environment and have a "homepage", I configured a custom dashboard with links and statuses for my services I host. I utilized homepage to help build this out

Network Routing & Security

Rather than exposing ports directly, all web traffic routes through Traefik. By combining Traefik with static IP routing in Technitium DNS, every Proxmox container and Docker service resolves securely via clean domain names. There is also a OPNsense firewall that all traffic runs through

Local AI

Utilizing GPU passthrough I am able to selfhost my own ai models as well as support training local models on the server allowing my regular pc to use its own graphics card for other purposes
